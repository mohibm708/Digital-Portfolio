What Is a Footnote?

Footnotes are used in almost every kind of informative media, like articles, books, or entries. Their main purpose is to provide supplementary information or a citation in a way that doesn't make the text cluttered or hard to understand, kinda like a more discreet pair of parentheses.

Digitally, footnotes are an alternative to the "title" attribute, which have nearly identical functions. However, there's one detail that sets the two apart: activation. If you open something like a Wikipedia article and hover over the reference to a footnote, you may be presented with the same information written in the "Footnotes" section. That's the only similarity that references share with the "title" attribute, though, and that becomes a problem for mobile browsers. Why? The "title" attribute can only be accessed through hovering, and you can't really hover on a phone. The reference, though, can be both hovered over and clicked to access the annotation.

So which one's better? It depends, really. Footnotes are more mobile-friendly, but if you need to apply more information to something like an image or a link, the "title" attribute would do the job in a more visually appealing way. Additionally, footnotes are better used when the added details are relevant and helpful, so if you wanted to hide a joke within your code, it'd be better to write it into the "title" attribute.

How to Create a Footnote

Creating a footnote is fairly easy. The process itself has two stages: the reference itself and the footnote that it links to. Most footnotes and their sections have links that redirect to one another so that the reader may easily continue reading, meaning that each will have to consist of two things:

1. A link to the destination.

2. An "id" attribute, which resembles the "class" attribute but can only be used once. This allows elements to be used as "href" destinations to certain parts of text.

As long as those two are present in both the redirect and the footnote, any further additions or changes serve a mainly aesthetic use.

The model for a reference and footnote that link to one another, in its most basic form, would resemble:

Reference: <p>This page took a long time to create. <a href="#footnote1" id="reference1">[1]</a></p>

Footnote: <p><a href="#reference1" id="footnote1">1.</a> In fact, it took about 53 minutes.</p>

In action, this code would look like this:


This page took a long time to create. [1]


Once you've got this model done, you can add your own personal tweaks, like using <sup> tags in the reference to make it easier to discern from the rest of the text, like this:
